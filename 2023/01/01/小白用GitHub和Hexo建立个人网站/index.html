小白用GitHub和Hexo建立个人网站

2023-01-01
  • Git使用

请结合Hexo官网文档阅读本文章。
Hexo中文官网：https://hexo.io/zh-cn/。

浅谈Git，GitHub和Hexo

这个网站建立的主要原理是使用GitHub的服务器来存放网站，而网站框架由Hexo来实现。

先简单说说Git，它是用来做程序版本管理的一个小工具，在程序开发中必不可少。用Git的人多了以后，出现了一个叫GitHub的线上社区，大家可在里面分享自己的由Git管理的代码。

JavaScript脚本语言，常用于网页开发。Node.js是JavaScript的一种运行环境，而Hexo是基于Node.js的程序软件，该程序致力于个人博客网站的搭建。 # 关于GitHub的准备

进入GitHub官网，按照提示注册一个账号。GitHub官网地址：https://github.com/。由于它是国外网站，经常会有连接不稳定情况，请多次尝试，你也可以查一下解决链接不稳的办法（本人亲测了改host文件的办法，要经常刷新，而且不是百试百灵）。

然后在自己账号下创建一个新的仓库（repository），并建立对应的README.md文件。仓库名字要起为“<Github用户名>.github.io”，便于之后Hexo识别。比如我的用户名是“realyezhihzo”，那么仓库名要为“realyezhihao.github.io”。

关于Git的准备

下载

在官网下载Git。Git官网网址：https://git-scm.com/。

安装

下载完成后，打开安装包。大多数地方按照默认设置一直点“Next”即可，但在“Adjusting the name of the initial branch in new repositories”一页中，建议选择“Override the default branch name for new repositories”，并把名称改为“main”。

这最后一步经常会出现错误。以本人经验，如果用本地服务器也无法访问，那应该是做的网站本身有错导致静态页面编译生成时出现了错误，这时就应该回去检查网站设置、页面文件、页面主题等等是否有语法逻辑的错误。如果本地服务器访问正常但部署失败，大多是因为Git和GitHub之间连接不够畅通。可使用“ssh git@github.com”来检测。检测正常了但还是出错（我经常遇到，已经是不明所以了），只能推测是人品不好:-(，等待一会儿后再尝试。
